12 **Backport** is an ongoing effort to bring modern C++ utilities to be compatible
13 with C++11.
14 
15 It is a header-only library that implements much of the functionality from
16 C++14, C++17, and C++20 to be compatible with C++11. `constexpr` support is
17 conditionally enabled based on the language setting of the compiler (e.g.
18 if using C++14, more `constexpr` is available to the user than with C++11)

37 ## Rationale
38 
39 C++11 was release _9 years ago_, and yet many organizations still have yet to
40 move onto more modern releases. In some cases this is due to security reasons,
41 in others it may be that the toolchains don't yet exist for more modern C++
42 support. In any case, much of the newer C++ library types don't actually
43 require newer C++ language features -- but provide significant value to the
44 developer by creating a new vocabulary that doesn't exist in prior versions.
45 
46 Much of the newer utilities may also have parallels in Boost libraries,
47 however many organizations are unable to adopt this -- either due to the domain
48 being too constrained, or due to Boost being a large dependency to introduce.
49 
50 As a result, this library exists as a standalone, **header-only**,
51 dependency-free drop-in for newer standard library features from C++11, C++14,
52 C++17, and C++20.

197 ### Where is `std::filesystem`?
198 
199 This library aims to deliver a _header-only_ solution to the many missing
200 types that have been introduced in newer standard libraries. `std::filesystem`
201 is a bit complicated to handle, since it introduces a dependency on the
202 operating system being compiled against. Implementing `std::filesystem` will
203 generally require conditional compilation, and -- for best practices --
204 separate source files in order to insulate use of system headers (lest this
205 library introduce the perils of the `<windows.h>` macros). This would change the
206 library fundamentally so that it is no longer a header-only solution -- and this
207 is not desirable.
208 
209 Third-party implementations of `std::filesystem` should exist, which can be
210 independently used.
